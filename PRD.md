Product Goal
To provide a unified platform for SDU students to easily access, manage, and stay updated on campus events, club activities, and academic deadlines in one place.

Problem Statement
Students often struggle with finding relevant campus activities and deadlines, which can hinder their academic and social participation. This product aims to solve that by centralizing event management and providing real-time updates.

Target Audience
Primary Users: SDU students, especially newcomers who want to integrate into campus life.
Secondary Users: Club organizers and university administration for managing and monitoring events.

User Roles
Student: Views, joins, and tracks events and deadlines.
Admin: Manages event listings, club memberships, and sends notifications.

User Scenarios
Student: Wants to stay informed about upcoming events and deadlines.
Admin: Wants to manage student participation in clubs and monitor event engagement.

Functional Requirements
Student login and registration.
Event and deadline management.
Club membership requests and management.
Personalized notifications for events and updates.

Non-Functional Requirements
Performance: Events and notifications must load within 2-3 seconds.
Reliability: The platform must be available 99.9% of the time.
Scalability: The platform should be able to handle growing user numbers as the university grows.

MVP Scope
User authentication (login/registration).
Event management and filtering.
Personalized user dashboard.

Out-of-Scope Features
Social sharing of events.
Advanced event categorization by type, location, etc.

Acceptance Criteria
Event Management: Admin can add and update events, which appear on the user dashboard.
User Authentication: Users can register, log in, and access their personalized event list.
